# EMS Game

A Godot 4.6 game project with automated WSL/Linux development environment setup using Nix.

## Quick Start

### A) Windows (via WSL)

This creates a clean Ubuntu 24.04 WSL distro with everything pre-configured:

```powershell
irm https://raw.githubusercontent.com/justinmoreira/EMS_Game/dev_setup/scripts/wsl/install.ps1 | iex
```

**What this does:**
1. Downloads Ubuntu 24.04 WSL image
2. Creates `ems-wsl` distro with systemd enabled
3. Installs Godot for Windows (or finds existing installation)
4. Clones project to `C:\Users\<you>\EMS_Game` (accessible by both Windows and WSL)
5. Installs Nix, Docker, direnv, and all dev dependencies
6. Configures environment to auto-load on `cd`

**After installation:**
```powershell
wsl -d ems-wsl
cd /mnt/c/Users/<you>/EMS_Game
```

### B) Linux / Existing WSL

```bash
curl -fsSL https://raw.githubusercontent.com/justinmoreira/EMS_Game/dev_setup/scripts/install.sh | bash
```

Installs to `~/EMS_Game` with the same dependencies as above.

## Development Workflow

The project uses [just](https://just.systems/) as a task runner. All commands work from WSL:

```bash
just          # List all available commands
just --choose # TUI choose commands
just edit     # Open Godot editor (Windows GUI from WSL)
just build    # Export game for web
just run      # Build + serve on localhost:8080
just stop     # Stop web server
```

**How it works:**
- **WSL**: Editor opens Windows Godot.exe via PowerShell, builds use Linux headless binary
- **Native Linux**: Both editor and builds use the native Linux binary
- Project files live in Windows filesystem (`/mnt/c/`) so Godot Windows can access them

## Project Structure

```
EMS_Game/
├── godot/              # Godot project files
├── server/             # Nginx config + web build output
│   ├── public/         # Built game (index.html, .wasm, .pck)
│   └── nginx.conf      # Server configuration
├── scripts/            # Installation scripts
│   ├── install.sh      # Project setup (Linux/WSL)
│   └── wsl/            # WSL-specific automation
└── Justfile            # Task definitions
```

## Environment Variables

Create `.env` in project root (auto-generated by installer):

```bash
GODOT_WIN=/mnt/c/path/to/Godot_v4.6-stable_win64.exe  # WSL only
```

## Uninstall / Cleanup

```powershell
# Remove WSL distro (keeps project files)
irm https://raw.githubusercontent.com/justinmoreira/EMS_Game/dev_setup/scripts/wsl/uninstall.ps1 | iex

# Manually delete project
rm -rf C:\Users\<you>\EMS_Game
```

**Other WSL commands:**
```powershell
wsl -d ems-wsl           # Specify if you have other distros
wsl --terminate ems-wsl  # Stop distro
wsl -l -v                # List all distros
```
