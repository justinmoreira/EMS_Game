# EMS Game

A Godot 4.6 game project with automated WSL and Linux development environments.

## Quick Start

### A) Windows (via WSL)

This creates a clean Ubuntu 24.04 WSL distro with everything pre-configured:

```powershell
irm https://raw.githubusercontent.com/justinmoreira/EMS_Game/main/scripts/wsl/install.ps1 | iex
```

**What this does:**
1. Downloads Ubuntu 24.04 WSL image
2. Creates `ems-wsl` distro with systemd enabled
3. Installs Godot for Windows (or finds existing installation)
4. Clones project to `C:\Users\<you>\EMS_Game` (accessible by both Windows and WSL)
5. Installs Nix, Docker, direnv, and all dev dependencies
6. Configures environment to auto-load on `cd`

**After installation:**
```powershell
wsl -d ems-wsl
cd /mnt/c/Users/<you>/EMS_Game
just code  # Open VSCode editor (If available)
```

### B) Linux / Existing WSL

```bash
curl -fsSL https://raw.githubusercontent.com/justinmoreira/EMS_Game/main/scripts/install.sh | bash
```

Installs to `./EMS_Game` (current directory) with the same dependencies as above.

## Development Workflow

The project uses [just](https://just.systems/) as a task runner. All commands work from WSL:

```bash
just                 # List all available recipes

just --choose        # Interface to select from commands

just edit            # Open the Godot editor (GUI; cross-platform)

just dev             # THE MAIN COMMAND
# Start Astro dev server & auto-rebuild Godot on git-tracked file changes

just build           # Full production build pipeline: Godot → Astro → server/public/

just run             # Build for Production and serve (Godot + Astro + Docker)
just stop            # Stop the Production Docker web server
just github-auth     # Authenticate with GitHub CLI and configure Git
just code            # Open VS Code workspace

# And other less important commands
```

## Project Structure

```
EMS_Game/
├── godot/              # Godot project files
├── server/             # Nginx config + web build output
│   ├── public/         # Built game (index.html, .wasm, .pck)
│   └── nginx.conf      # Server configuration
├── scripts/            # Installation scripts
│   ├── install.sh      # Project setup (Linux/WSL)
│   └── wsl/            # WSL-specific automation
└── Justfile            # Task definitions
```

## Environment Variables

Create `.env` in project root (auto-generated by installer):

```bash
GODOT_WIN=/mnt/c/path/to/Godot_v4.6-stable_win64.exe  # WSL only
```

## Uninstall / Cleanup

```powershell
# Remove WSL distro (keeps project files)
irm https://raw.githubusercontent.com/justinmoreira/EMS_Game/main/scripts/wsl/uninstall.ps1 | iex

# Manually delete project
rm -rf C:\Users\<you>\EMS_Game
```

**Other WSL commands:**
```powershell
wsl -d ems-wsl           # Specify if you have other distros
wsl --terminate ems-wsl  # Stop distro
wsl -l -v                # List all distros
```

### Technologies
- `nix flakes`: Replicable environments between dev and prod
- `direnv`: auto-load environments on `cd`
- `Just`: `Make` alternative for reusable recipes/commands
- `Docker`: VM for isolated webserver
- `nginx`: Router/Webserver
- `godot`: GUI for game editing and binaries for compilation to web
